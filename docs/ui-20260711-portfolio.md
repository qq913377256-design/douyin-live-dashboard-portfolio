# 抖音直播电商看板在线作品页UI规格

日期：2026-07-11

## 1. 设计目标

- 页面面向数据分析、直播运营、产品与技术岗位面试官，首屏10秒内回答项目是什么、作者负责什么、数据边界是什么
- 视觉方向为暖色编辑部文档页，克制、清晰、带少量纸张质感，不做营销落地页，不使用大面积渐变、玻璃拟态、霓虹光效和装饰性3D图形
- 页面记忆点是浅色说明文档中嵌入一块完整的深色实时看板，形成文档解释层与产品证据层的明确对比
- 参考图只借用固定侧目录、宽松留白、白色浅卡片和细分割线，不复制参考图中的图标、文案、插画和下载卡片结构

## 2. 页面骨架

### 2.1 桌面布局，视口宽度大于等于1200px

- 左侧目录固定在视口，宽248px，高100vh，背景`#FBF5EC`，右边框1px `#E4D2BD`
- 目录内容左右24px，上24px，下32px，内部可独立滚动，滚动条宽6px
- 主内容区左外边距248px，最小宽度0
- 正文容器最大宽度1120px，左右内边距48px，顶部56px，底部96px，水平居中
- Hero最大宽度960px并居中，后续章节卡片宽度100%
- 章节间距28px，卡片内部桌面端48px

### 2.2 中等桌面布局，1024px至1199px

- 左侧目录宽224px
- 主内容区左外边距224px
- 正文左右内边距32px，卡片内部36px
- 双列内容网格允许切为单列，功能卡片仍可保持两列

### 2.3 平板布局，768px至1023px

- 不保留固定侧栏，顶部显示固定目录栏，高64px，背景使用带轻微透明度的`rgba(251, 245, 236, 0.94)`，配合`backdrop-filter: blur(12px)`，不支持时退化为纯色
- 顶栏左侧显示项目短名直播健康诊断，右侧为目录按钮
- 正文顶部留出88px，左右内边距24px，卡片内部28px
- 目录采用右侧抽屉，宽度`min(360px, 88vw)`

### 2.4 手机布局，小于768px

- 顶栏高58px，正文顶部78px，左右内边距16px，底部72px
- 卡片圆角14px，内部20px，章节间距18px
- 目录抽屉宽度`min(336px, 90vw)`，从右侧进入
- Hero不使用满屏高度，保证首屏能看到标题、摘要、主CTA和项目状态
- 所有多列网格改为单列，按钮组占满可用宽度
- 390px视口下页面本体不得横向滚动，只有带`.table-scroll`的表格容器允许横向滚动

## 3. 视觉变量

| `--surface` | `#FFFCF8` | 章节卡片 |
| `--surface-strong` | `#FFFFFF` | 表格、截图外框、弹层 |
| `--ink` | `#332B25` | 主正文，避免纯黑 |
| `--ink-soft` | `#6D6055` | 辅助说明 |
| `--heading` | `#765735` | 一级至三级标题 |
| `--line` | `#E3CEB7` | 边框和分割线 |
| `--accent` | `#168F91` | 主按钮、链接、目录选中线 |
| `--accent-hover` | `#0F7476` | 主按钮hover |
| `--accent-soft` | `#DDF1EF` | 标签和提示背景 |

| `--focus` | `#006F73` | 键盘焦点环 |

- 正文与背景对比度至少4.5:1，大字号标题至少3:1
- 青绿色只用于可交互元素、状态标记和少量关键数字，不把整页变成青绿色主题
- 棕色只承担文档标题和分隔层级，不用于小字号正文

### 3.2 字体

- 标题字体栈：`"Noto Serif SC", "Source Han Serif SC", "Songti SC", STSong, serif`
- 正文字体栈：`"Noto Sans SC", "Source Han Sans SC", "Microsoft YaHei", "PingFang SC", sans-serif`
- 不从第三方CDN加载字体，首版使用本机字体栈，避免字体阻塞和GitHub Pages跨域依赖
- Hero标题桌面端52px、行高1.18、字重700、字间距`-0.02em`
- Hero标题1024px为44px，手机为34px
- 章节标题桌面端30px，手机24px，行高1.3
- 卡片标题20px，手机18px
- 正文16px，行高1.78，手机15px，行高1.72
- 辅助说明14px，行高1.65，表格13px至14px，行高1.55
- 数字指标和阶段编号使用`font-variant-numeric: tabular-nums`

### 3.3 圆角、阴影和边框

- 主章节卡片圆角18px，边框1px `var(--line)`
- 卡片阴影为`0 12px 36px rgba(82, 61, 40, 0.07)`，hover不抬升，避免整页漂浮
- 小卡片圆角12px，阴影取消，使用边框区分层级
- 按钮圆角999px，输入控件类元素本页不存在
- 深色截图外层圆角18px，内层圆角12px，边缘不裁切关键看板内容

### 3.4 背景细节

- 页面背景使用纯CSS纸张纹理，叠加两层低透明径向渐变，透明度不超过3%，不使用外部纹理图片
- Hero右上角可放一个由细线和圆点组成的数据脉冲装饰，使用CSS绘制，透明度12%，不使用通用AI星芒和漂浮球
- 所有背景装饰设置`pointer-events: none`并从可访问树隐藏

### 4.1 桌面目录

- 顶部显示小号项目标记`LIVE OPS / PORTFOLIO`，字距0.12em，字号11px，颜色`var(--accent)`
- 下方显示目录标题，20px标题字体，底部有40px长的棕色细线
- 导航项高44px，左右12px，圆角10px，文字14px
- 每项左侧使用两位章节编号`01`至`11`，不使用emoji和业务无关图标
- 默认文字`var(--ink-soft)`，hover背景`rgba(227, 206, 183, 0.28)`，文字`var(--heading)`
- 当前章节背景`var(--accent-soft)`，文字`#0B6668`，左侧3px青绿色指示线
- 当前项同时设置`aria-current="location"`，不能只依赖颜色表达状态
- 目录底部常驻非官方说明，字号12px，行高1.55，与导航间隔24px

### 4.2 移动目录抽屉

- 打开按钮可触区域至少44×44px，图标为三条水平线加文字目录
- 抽屉遮罩使用`rgba(35, 29, 24, 0.42)`，点击遮罩关闭
- 抽屉打开后焦点移到关闭按钮，Tab焦点限制在抽屉中，Esc关闭，关闭后焦点回到打开按钮
- 点击导航项后更新hash、关闭抽屉，再将对应章节标题聚焦到可见位置
- 页面滚动在抽屉打开期间锁定，关闭时恢复原滚动位置
- 抽屉过渡时长220ms，使用`transform`，减少动态偏好下取消位移动画

## 5. Hero

### 5.1 内容顺序

1. 眉题：个人项目、实时运营分析原型
2. 主标题：抖音直播电商健康诊断看板
3. 副标题：实时观察在线人数、评论互动、商品切换和问题弹幕，把直播间数据转成可执行的运营建议
4. 标签：数据分析、实时监控、直播运营、个人项目
5. 三项项目元信息：状态、数据范围、我的角色
6. CTA组：观看60秒演示、查看指标口径、查看路线图
7. 首屏数据与隐私声明

### 5.2 排版与组件

- Hero使用单栏偏左排版，不居中堆叠，保证长中文标题和元信息更易扫读
- Hero顶部留白桌面端48px，底部36px
- 标题最大宽度820px，副标题最大宽度760px
- 标签为细边框胶囊，背景透明，边框`#D7BFA3`，文字13px
- 元信息使用三列浅色信息带，桌面端每列最小180px，手机端改为单列
- 每项元信息包含11px大写或小号标签与15px正文，状态前增加8px实心圆点
- 主CTA为青绿色实心按钮，次CTA为白底棕色描边按钮，路线图为文本链接加向下箭头
- 按钮高度46px，左右20px，手机端主按钮和次按钮宽度100%
- 首屏声明使用左侧3px青绿色线条，不用警告图标，正文14px

## 6. 章节卡片

- 每个主章节使用`section`与可链接`id`，标题行由章节编号、标题和可选状态标签组成
- 章节编号使用12px青绿色等宽数字，不使用彩色emoji
- 标题下方可放一句60至100字的章节导语，最大宽度780px
- 章节标题与内容间距24px，内容块之间间距20px
- 标题锚点滚动需预留桌面24px、移动端82px，避免被固定顶栏遮挡
- 章节标题可在hover时显示链形锚点，但不改变标题位置，键盘聚焦时持续可见
- 卡片本身不做滚动入场动画，首屏仅允许Hero内容一次性淡入，避免长文阅读时持续晃动

## 7. 视频与深色截图

### 7.1 60秒视频

- 视频卡片位于章节白卡内部，使用深色衬底`var(--dashboard-matte)`和16:9比例容器
- 外框采用两层结构，外层白色留边12px与1px棕色边框，内层深色衬底，像作品集中的装裱画面
- 视频元素宽100%、`object-fit: contain`，不裁切深色看板边缘
- 封面图加载前提供与看板一致的深蓝占位，不出现白色闪烁
- 使用浏览器原生controls，`preload="metadata"`，默认静音，不自动播放
- 视频下方显示一行片段目录：全景、趋势、异常建议、评论意图、商品切换，桌面端横排，手机端横向滚动
- 片段目录是说明文字，不伪装成可点击时间轴，除非实现真实跳转
- 视频加载失败时保留16:9容器，显示视频暂不可用、可先查看下方看板截图，并提供指向截图章节的锚点
- 错误状态使用浅米色文字和青绿色链接，不显示浏览器默认破损图标
- 视频说明条写明使用合成演示数据、无真实账号和直播链接

### 7.2 看板截图

- 截图使用`dashboard-current.webp`，懒加载，必须来自合成演示数据
- 桌面端完整展示16:9截图，点击可打开原尺寸预览弹层
- 预览弹层背景`rgba(11, 18, 32, 0.92)`，最大宽度96vw、最大高度92vh，图片`object-fit: contain`
- 弹层支持Esc关闭、点击遮罩关闭、关闭按钮、焦点锁定和焦点返回
- 手机端不把完整截图硬缩到无法阅读，默认展示完整缩略图，下面提供查看大图按钮，弹层内允许双指缩放依赖浏览器原生图片缩放
- 截图下方使用短图注说明蓝色趋势图、粉色评论信号和绿色商品状态分别表达什么
- 不对截图增加高饱和外发光，使用细线和留白与页面衔接

## 8. 核心内容组件

### 8.1 功能模块卡片

- 桌面端两列，平板和手机单列
- 每张卡包含模块名、回答的业务问题、关键展示、运营动作
- 模块名20px，业务问题使用棕色文字，运营动作放在青绿色浅底区域
- 不用统一插画，卡片左上使用`01`至`10`编号形成连续阅读节奏

### 8.2 数据范围双栏

- 桌面端分为当前可观测与当前不可获得两栏，比例1:1
- 可观测栏使用青绿色细顶边，不可获得栏使用棕色细顶边
- 每项前使用简洁CSS圆点，不使用对勾或叉号暗示价值判断
- 两栏后紧接非官方与隐私声明，声明不可被折叠隐藏

### 8.3 指标表格

- 表头背景`#F3E7D8`，文字`var(--heading)`，字重650
- 行高至少52px，奇偶行不使用强烈斑马纹，hover仅加`#FBF6EF`
- 表格最小宽度860px，外层横向滚动，容器顶部和底部提供细边框
- 首列指标名在滚动时保持sticky，背景随所在行变化，右侧有1px分隔线
- 小屏在表格上方显示左右滑动查看全部口径，使用纯文字提示
- 公式使用等宽字体栈`"SFMono-Regular", Consolas, "Liberation Mono", monospace`，背景`#F5EEE5`，圆角6px，允许换行

### 8.4 异常与建议卡片

- 每张卡必须包含级别、触发原因、关键数据、建议动作、发生时间
- 级别色只用于左侧4px标记和小标签，正文保持深色
- 紧急`var(--danger)`、重要`var(--warning)`、提醒`var(--accent)`、机会`var(--success)`、系统`var(--future)`
- 建议动作区域使用浅底色和动词开头，避免做成聊天气泡或AI助手样式

### 8.5 运营分析方法

- 使用现象、可能原因、下一步验证、运营动作四列逻辑链
- 桌面端用四列箭头流程，1024px以下改为纵向卡片
- 箭头只在桌面端显示，使用1px细线，不使用大图标
- 每条逻辑链是独立`article`，屏幕阅读器按完整案例读取

### 8.6 直播复盘报告，Phase 2

- 章节标题右侧放规划中标签，背景`#F7E9D7`，文字`#8B541B`
- 卡片顶部明确写明当前状态：规划中，尚未作为现成功能开放
- 数据流以五步横向流程展示：采样数据、事件归因、场次摘要、行动建议、Markdown、PDF报告
- 小屏改为纵向编号列表
- 预期报告结构使用纸张样式预览，白底、轻阴影、左侧装订线，不伪造已生成的报告截图

### 8.7 路线图

- 使用阶段卡片，不做横向超长时间轴
- 桌面端三列，Phase 1单独占第一行并标记已完成，Phase 2与Phase 3占第二行，后续阶段依次排列
- 状态标签统一为已完成、规划中、后续设想，不用模糊的进行中百分比
- 已完成卡顶部绿色，规划中卡顶部橙色，后续设想卡顶部灰紫色
- 路线图前加一句目标说明，强调直播中发现问题、直播后验证原因、下一场形成改进动作的闭环

### 8.8 技术实现

- 使用轻量架构图表示采集层、存储层、分析层、API层、展示层
- 在线作品页单独画在展示层外侧并标明静态站点、无实时API连接
- 架构节点使用白底边框卡片，连接线用棕色和青绿色，不做代码编辑器风格
- 技术标签不使用品牌彩色Logo，统一使用文字胶囊

### 8.9 FAQ

- 使用原生`details`与`summary`，默认展开第一项
- 每项上下内边距18px，条目间用1px分割线
- 展开图标为CSS绘制加号，展开后旋转45度
- `summary`焦点态完整可见，点击区域覆盖整行

## 9. 按钮、链接和状态

- 所有交互目标最小44×44px，文本链接在正文中允许高度随行高，但需有清晰下划线
- 主按钮hover改为`var(--accent-hover)`并上移1px，active恢复原位
- 次按钮hover背景`#F3E7D8`，active背景`#ECDAC5`
- 文本链接默认青绿色并有1px下划线，hover下划线厚度2px
- `focus-visible`统一使用3px `rgba(0, 111, 115, 0.34)`外环与2px白色间隔
- 禁用状态不作为首版常规状态，视频加载期间可显示`aria-busy="true"`
- 外链若未来加入，需要显示外链标识并写明新窗口行为，首版CTA全部为站内锚点

## 10. 动效

- Hero首次加载允许执行一次240ms淡入与8px上移，标题、副标题、CTA延迟各40ms，总时长不超过400ms
- 目录选中背景150ms，按钮颜色和位移120ms，抽屉220ms
- 截图弹层淡入160ms，图片不缩放弹跳
- 页面滚动使用`smooth`，若`prefers-reduced-motion: reduce`则改为`auto`
- 减少动态偏好下关闭Hero位移、抽屉位移、按钮上移和所有非必要transition
- 不实现滚动视差、持续漂浮、数字滚动计数和整页卡片逐个飞入

## 11. 无障碍

- 页面包含跳到正文链接，键盘首次Tab时出现
- 使用`header`、`nav`、`main`、`section`、`footer`语义结构，每个section由可见标题标记
- 目录`nav`提供`aria-label="页面目录"`，移动抽屉按钮同步`aria-expanded`与`aria-controls`
- IntersectionObserver只更新目录状态，不抢焦点，不在用户滚动时改写焦点
- hash直达章节时，标题可程序化聚焦但不显示重复视觉变化
- 视频必须有字幕轨道或内嵌中文字幕，若使用内嵌字幕，在视频旁提供等价文字分镜摘要
- 所有图片填写描述用途的alt，纯装饰图使用空alt
- 状态不能只靠颜色，必须同时显示文字标签
- 表格使用`caption`、`thead`、`th scope`，滚动容器可键盘聚焦并标注可横向滚动
- 弹层使用`role="dialog"`、`aria-modal="true"`与可见标题
- 触控端不依赖hover显示关键内容

## 12. 响应式验收矩阵

## 13. 返回顶部与页尾

- 返回顶部按钮在滚动超过首屏后出现，桌面端固定右下24px，移动端右下16px并避开系统安全区

- 页尾显示项目类型、技术栈、最后更新时间和非抖音官方说明

## 14. 实现边界
