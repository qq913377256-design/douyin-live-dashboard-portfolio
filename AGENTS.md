# 抖音直播电商看板在线作品页规范

## 项目定位

- 本项目是独立的静态作品页，用于展示直播健康诊断看板的项目背景、分析方法、数据边界和路线图。
- 技术栈为Vite、React、TypeScript和原生CSS，不连接本地采集服务，不提供登录、搜索或后台功能。
- 项目独立初始化Git，只发布本目录内容，不提交相邻的采集器项目或资料包。

## 视觉与内容

- 采用暖米色文档背景、白色浅卡片、棕色标题和青绿色强调色。
- 参考图只用于侧目录、留白和卡片层级，不复制其业务内容、图标或文案。
- 现有深色看板作为核心展示画面；公开截图和视频必须来自合成演示数据。
- 内容集中维护在`src/content.ts`，页面组件不得散落重复业务文案。
- 已完成、规划中和后续设想必须明确区分，不把规划能力写成现成功能。

## 安全边界

- 禁止提交Cookie、token、签名参数、真实直播间数字链接、账号信息、用户标识、本地数据库和绝对路径。
- 禁止把本地采集服务部署或代理到公开站点。
- `docs-layout-reference.png`和用户提供的原始看板截图只作本地参考，不进入本项目。
- 公开资产只允许使用脱敏后的`dashboard-demo.mp4`、`dashboard-poster.webp`和`dashboard-current.webp`。
- 页面必须声明非抖音官方产品，指标属于采样值、代理指标、规则结果或模型分类，不等同于官方后台数据。

## 工程约定

- 源码放在`src/`，公开素材放在`public/assets/`，项目文档放在`docs/`。
- GitHub Pages基础路径固定为`/douyin-live-dashboard-portfolio/`。
- 桌面端固定侧目录，移动端使用可关闭目录抽屉；交互支持键盘、焦点管理和减少动态效果。
- 宽表格在移动端横向滚动，页面本身不得产生横向溢出。
- 所有按钮和链接提供hover、focus-visible和active状态。
- 不引入组件库，避免与单页文档规模不匹配的依赖。

## 验证

- 每次非平凡改动执行类型检查和生产构建。
- 使用Playwright检查1440、1024、768和390像素视口、目录交互、锚点、视频状态与控制台错误。
- 发布前扫描公开目录中的敏感信息和本地绝对路径。
- 使用ffprobe确认演示视频时长、尺寸、编码和文件大小。

非平凡改动追加到`docs/WORKLOG.md`，标题格式为`## YYYY-MM-DD HH:MM`，写明完成内容、验证结果、公开链接和未完成项。
